考核任务：打造志愿者系统 2.0
我们将重构第一周的项目，解决两个核心痛点：程序重启后数据丢失，以及程序无法与互联网交互。

具体需求如下：

数据持久化 (赋予程序“记忆”)
程序启动时，需要检查当前目录下是否存在 data.json 文件。
如果文件存在，就从中读取数据，加载到内存中的 volunteers 列表里。
如果文件不存在，则初始化一个空的 volunteers 列表。
功能升级：
在原有的“添加志愿者”功能中，每成功添加一个新成员后，必须立即将更新后的整个 volunteers 列表写入到 data.json 文件中。
这意味着，当你关闭并重新运行程序时，之前添加的志愿者信息依然存在！
网络功能集成 (连接互联网)
在主菜单中增加一个新选项，例如：6. 获取每日一句，用来鼓舞一下志愿者。
当用户选择该选项时，你的程序需要：
使用 requests 库向一个公开的 API 发送 GET 请求。推荐 API：https://v1.hitokoto.cn/ (一言，随机返回一句正能量的话)。
成功获取响应后，解析返回的 JSON 数据，并将其中的句子内容打印在控制台上。
（可选加分） 尝试处理可能出现的网络错误（比如断网），如果请求失败，给出一个友好的提示，而不是让程序崩溃。
🌟 加分项
代码封装：将文件读取、文件写入、网络请求等功能封装成独立的函数，让主逻辑更清晰。⭐
完善的 requirements.txt：确保你的 requirements.txt 文件包含了所有必要的库（主要是 requests）。⭐⭐
错误处理：对文件读写、网络请求等可能出错的地方，使用了 try...except 语句进行优雅的错误处理。⭐⭐⭐
附录：代码提交指南 (拥抱 GitHub)
上一次的进阶任务是使用 Gist 提交代码。 从本次考核开始，我们将使用 GitHub 仓库提交代码。

方式一：网页版提交 (推荐新手)
这种方式最简单，无需安装任何额外软件，像使用网盘一样直观。

注册/登录 github.com。
创建新仓库 (New Repository)：点击右上角 + 号，给仓库起个名字 (如 volunteer-system-v2)，设为 Public，不要勾选任何初始化选项，直接创建。
上传文件：在空仓库页面，点击 uploading an existing file 链接。
拖拽文件：将你本地的 main.py 和 requirements.txt 等文件拖拽到上传区域。
注意：绝对不要上传 venv 文件夹！

提交更改 (Commit changes)：在页面下方填写一句提交描述（如“完成第二次考核任务”），然后点击绿色按钮确认。
提交链接：将你的 GitHub 仓库地址（URL）作为作业提交即可。
方式二：命令行 Git 提交 (进阶挑战)
如果你对自己的学习能力有信心，或者想体验专业开发者的工作流程，强烈推荐尝试使用命令行 Git！它能让你更深刻地理解版本控制。

基本流程：

安装 Git：从 git-scm.com 下载并安装。

本地初始化：在你的项目文件夹中，打开终端，运行 git init。

创建 .gitignore 文件：新建 .gitignore 文件，并写入 venv/ 和 __pycache__/ 来忽略不必要的文件。

添加与提交：

git add .
git commit -m "你的提交信息"
连接远程仓库并推送：

git remote add origin <你的仓库HTTPS链接>
git push -u origin main
提交链接：同样，将你的 GitHub 仓库地址作为作业提交。

Git 的学习需要一个过程，如果遇到问题，可以在群里提问或查阅相关教程。
